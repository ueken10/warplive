# 3Dアバターアシスタント「WarpLive」仕様書

## 1. プロジェクト概要

Gemini Live APIと連携し、音声で対話できる3D VRMアバターアシスタントのWebアプリケーション。

アバター名「浅香リリ」が、音声認識・音声合成・感情表現・アニメーションを組み合わせ、ユーザーと自然な対話を行う。

### 1.1 技術方針（AGENT.md準拠）

| 項目 | 方針 |
| ------ | ------ |
| 言語 | **Vanilla JavaScript**（`.js`拡張子）。TypeScript不使用。型情報はJSDocで記述 |
| ビルドツール | **不使用**（Vite / Webpack / ESBuild等すべて排除） |
| モジュール方式 | ブラウザネイティブ ES Modules（`<script type="module">` + `import/export`） |
| 外部ライブラリ | CDN（esm.sh / jsdelivr）から直接import、またはビルド済みファイルの相対パス参照 |
| 実行環境 | 事前コンパイル不要。最新ブラウザのJSエンジンでそのまま実行 |

### 2.1 アバター表示

### 2.2 音声対話（Gemini Live API）

#### 2.2.1 接続方式

| 項目 | 仕様 |
| ------ | ------ |
| API | Gemini Live API（`wss://generativelanguage.googleapis.com/ws/google.ai.generativelanguage.v1alpha.GenerativeService.BidiGenerateContent`） |
| 認証 | URLパラメータ `?key=API_KEY`。APIキーはユーザーがUIで入力（ローカルストレージに保存） |
| モデル | `gemini-2.0-flash-live-001`（または最新のLive API対応モデル） |
| 音声入力 | 16kHz, 16-bit PCM, mono（ブラウザマイク → MediaRecorder / AudioWorklet でPCM化） |
| 音声出力 | Live APIから受信したPCMチャンクをAudioBufferに変換して再生 |
| APIキー警告 | ハンバーガーメニューのAPIキー設定項目の上に「APIキーはローカルでのみ使用し、公開しないでください」の警告を表示 |

#### 2.2.2 ウェイクワード（方式B: ローカル検出 + オンデマンド接続）

| 項目 | 仕様 |
| ------ | ------ |
| 検出方式 | Web Speech API `SpeechRecognition`（`continuous: true`, `interimResults: true`）で常時ローカル認識 |
| ウェイクワード | 選択中のアバターに応じて切り替え（下表参照） |
| 検出後の動作 | ① Live APIセッションを開始 ② ウェイクワード以降の発話をバッファリングしてLive APIに送信 |
| 一時応答 | ウェイクワード検出直後、Live API接続完了を待たずに `SpeechSynthesis` で短い一時応答（「はい」等）を即座に発話。アバターは `[greeted]` 表情 + VRMA_02 (Greeting) を再生。Live API接続完了後にユーザー発話の送信を開始。一時応答の発話中にLive API音声が再生されないようミュート制御 |
| 連続対話 | ウェイクワード検出後、**アイドル8秒間**はLive APIセッションを維持。新たな発話があれば延長 |
| セッション切断 | アイドル8秒経過でLive APIセッションを切断し、ウェイクワード待ちに戻る |
| 無料枠配慮 | 常時接続せず、ウェイクワード検出時のみ接続するためAPI消費を最小化 |
| フォールバック | Web Speech API非対応ブラウザでは、タップ/ボタン押下でセッション開始するフォールバックUIを提供 |

**アバター別ウェイクワード:**

| アバターファイル | 名前 | ウェイクワード |
| ------------------ | ------ | ---------------- |
| `asaka_lily.vrm` | 浅香リリ | 「ねえ、リリ」「Hey Lily」 |
| `matsuda_emma.vrm` | 松田エマ | 「ねえ、エマ」「Hey Emma」 |
| `miura_luca.vrm` | 三浦ルカ | 「ねえ、ルカ」「Hey Luca」 |

※カスタムVRMインポート時はデフォルト（リリ）のウェイクワードを使用。

**シーケンス図:**

```
[常時] Web Speech API でローカル音声認識中
         │
         ├─ ウェイクワード検出（例: 「ねえ、リリ」）
         │     │
         │     ├─ 一時応答（SpeechSynthesis で「はい」を即座に発話 + Greetingアニメーション）
         │     │
         │     ├─ Live API WebSocket接続開始
         │     ├─ ウェイクワード以降の発話をLive APIに送信
         │     ├─ Live APIからの音声応答を再生 + リップシンク
         │     │
         │     └─ アイドル8秒間維持
         │           ├─ 新たな発話あり → セッション延長
         │           └─ アイドル8秒経過 → セッション切断 → ウェイクワード待ちに戻る
         │
         └─ ウェイクワード未検出 → 何もしない（API消費ゼロ）
```

#### 2.2.3 リップシンク 【スコープ外】

> **本バージョンでは実装しない。** Live API音声出力の再生のみ行い、VRM口形状（`aa`）へのマッピングは行わない。

| 項目 | 仕様 |
| ------ | ------ |
| 方式 | Live API音声出力を `AnalyserNode` で振幅解析 → VRM表情 `aa`（口を開ける）にマッピング |
| 音声フォーマット | Live API出力: 16kHz PCM → AudioContext サンプルレートにリサンプリング |
| マッピング | 振幅の絶対値 → `vrm.expressionManager.setValue('aa', value)` |
| スムージング | lerp（線形補間）でカクカクを防止。係数 0.3〜0.5 |
| クローズドマウス | 発話終了時は `aa` を 0 に徐々に戻す |

### 2.3 感情表現（タグ前置方式）

#### 2.3.1 感情タグ仕様

| 項目 | 仕様 |
| ------ | ------ |
| 方式 | AI応答テキストの先頭に `[emotion]` タグを付与 |
| 対応感情 | `neutral`, `joy`, `angry`, `sorrow`, `fun`, `surprised`, `greeted` |
| タグ形式 | 例: `[joy]こんにちは！今日もいい天気ですね。` |
| 抽出方法 | `output_audio_transcription` のテキストから正規表現 `^\[(\w+)\]` で抽出 |
| 字幕表示 | タグを除去したテキストを字幕として表示 |

#### 2.3.2 system instruction（感情タグ指示）

AIに対するsystem instructionに以下を含める：

```
感情表現として、応答テキストの先頭に必ず以下のいずれかのタグを付けてください。
タグは [neutral], [joy], [angry], [sorrow], [fun], [surprised], [greeted] のいずれかです。
タグは発声せず、テキストの先頭にのみ付けてください。
例: [joy]こんにちは！
```

#### 2.3.3 感情 → 表情・アニメーション マッピング

| 感情タグ | VRM表情 (blendShape) | VRMAアニメーション |
| ---------- | ---------------------- | ------------------- |
| `[neutral]` | neutral（デフォルト） | VRMA_06 (Model pose) |
| `[joy]` | happy | VRMA_03 (Peace sign) |
| `[angry]` | angry | 007_gekirei (激励＆激怒) |
| `[sorrow]` | sad | 005_smartphone (スマホを弄って拗ねる) |
| `[fun]` | relaxed | VRMA_03 (Peace sign) |
| `[surprised]` | surprised | 008_gatan (驚いてガタンと立ち上がる) |
| `[greeted]` | happy | VRMA_02 (Greeting) |

#### 2.3.4 アニメーション遷移

- 感情タグ検出時に対応するVRMAアニメーションを1ループ再生
- アニメーション終了後は待機アニメーション（2.3.5参照）に戻る
- VRMAの切り替えはクロスフェード（0.3秒）で滑らかに遷移

#### 2.3.5 待機アニメーション（アイドルモーション）

待機状態（ウェイクワード待機中・対話アイドル中）のアニメーションが単一（VRMA_01のみ）で生活感がない問題を改善するため、複数のモーションをランダム再生する。

| 項目 | 仕様 |
| ------ | ------ |
| 目的 | 単一の待機ポーズによる単調さを解消し、アバターが「その場で生活している」生活感を演出 |
| 対象状態 | ウェイクワード待機中、対話中のアイドル（ユーザー発話待ち・AI応答待ち） |
| 待機モーションプール | VRMA_01, VRMA_05, VRMA_07, 001_motion_pose, 003_humidai（計5種類） |
| 選択方式 | 各モーション終了時にプールからランダムに次のモーションを選択。直前と同じモーションは回避（連続重複防止） |
| 再生回数 | 各モーション1ループ再生。ループ終了後に次のモーションへ切替 |
| 切替タイミング | モーション終了時。クロスフェード（0.3秒）で滑らかに遷移 |
| 優先制御 | 感情タグによるアニメーション（2.3.3）が優先。感情アニメーション終了後に待機モーションに復帰 |
| 対話中の制御 | Live API音声再生中（リップシンク中）は待機モーションを継続。音声終了後に次の待機モーションへ切替可 |
| 設定 | 待機モーションプール・切替間隔は`config.js`で設定可能 |

| ~~`start_timer`~~ | 「タイマー」「時間を計って」「○分計って」 | **【スコープ外】** タイマーを開始。完了時に通知 |
| ~~`set_alarm`~~ | 「アラーム」「○時に教えて」「○時に起こして」 | **【スコープ外】** 指定時刻のアラームを設定。完了時に通知 |

> **本バージョンでは `get_current_time` のみ実装する。** それ以外の関数はスコープ外。

#### 2.4.2 各関数の実装詳細

> **【スコープ外】** `get_current_time` 以外の関数（`get_weather`, `start_timer`, `set_alarm`, `start_pomodoro`, `play_janken`）は本バージョンでは実装しない。以下は参考として残す。

##### **get_current_time**

- ローカルで `new Date()` を実行
- 結果をLive APIの `toolResponse` として返却
- AIが時刻を自然な日本語で発話

##### **get_weather**

- `navigator.geolocation.getCurrentPosition()` で現在地取得
- Open-Meteo API（無料・APIキー不要）`https://api.open-meteo.com/v1/forecast` にfetch
- 気象コード → 日本語天気描述に変換
- 結果を `toolResponse` として返却 → AIが天気を発話

##### **start_timer**

- パラメータ: `minutes`（分数。指定がない場合は質問する）
- 指定された分数（秒換算）の `setTimeout` を設定
- タイマー進行中はUIにカウントダウン表示
- 完了時の発話方式（ハイブリッド）:
  1. Live APIセッションが生存中 → `toolResponse`で完了を通知 → AIが「{minutes}分経ちました」と発話
  2. Live APIセッションが切断済み → `SpeechSynthesis` APIでフォールバック発話

##### **set_alarm**

- パラメータ: `time`（HH:MM形式）（指定がない場合は質問する）
- 現在時刻との差分を計算して `setTimeout` を設定
- 完了時の発話方式（ハイブリッド）:
  1. Live APIセッションが生存中 → AI経由で「HH時MM分ですよ」と発話
  2. Live APIセッションが切断済み → `SpeechSynthesis` APIでフォールバック発話
- 注意: ブラウザタブがバックグラウンド時のタイマー精度の低下に注意。`requestAnimationFrame`ではなく実時刻ベースで監視

##### **start_pomodoro**

- パラメータ: `cycles`（サイクル数。指定がない場合は1サイクル＝25分作業+5分休憩）
- 1サイクル = 作業25分 + 休憩5分。`cycles`回繰り返す
- 進行中はUIに現在のフェーズ（作業/休憩）と残り時間をカウントダウン表示
- 各フェーズ完了時の発話方式（ハイブリッド）:
  1. 作業25分完了 → Live APIセッションが生存中 → `toolResponse`で完了を通知 → AIが「25分経ちました。お疲れ様！休憩しましょう」と発話
  2. 休憩5分完了 → Live APIセッションが生存中 → AIが「休憩終了です。次の集中タイムいきましょうか？」と発話
  3. Live APIセッションが切断済み → `SpeechSynthesis` APIでフォールバック発話
- 全サイクル完了時 → AIが「ポモドーロ{cycles}サイクル完了！お疲れ様でした」と発話
- 注意: `start_timer`と同様、実時刻（`Date.now()`）ベースで監視

##### **play_janken**

- `Math.random()` で 0(グー) / 1(チョキ) / 2(パー) を選択
- AIの手を `toolResponse` として返却
- AIが結果を発話（「グー！...あなたの勝ち！」等）
- アバターは `[joy]` 表情 + VRMA_04 (Shoot) を再生

### 2.6 多言語対応

| 項目 | 仕様 |
| ------ | ------ |
| 対応言語 | 日本語（デフォルト）、英語、中国語（cmn-CN）、台湾華語（cmn-TW）、韓国語、マレー語、フィリピン語 |

### 2.9 能動的アクション 【スコープ外】

> **本バージョンでは実装しない。** アバターからの自発的発話・ユーザー存在検知・イベントトリガーは行わない。以下は将来拡張用の参考として残す。

ユーザーの発話を待たず、アバターから自ら話しかける機能。Gateboxの「能動的アクション」に相当し、一方通行の対話を防ぎ、キャラクターとの生活感を演出する。

#### 2.9.1 概要

| 項目 | 仕様 |
| ------ | ------ |
| 目的 | ユーザーからの一方的な話しかけのみならず、アバターから自ら話しかけることで「共に暮らす」体験を実現 |
| 発話方式 | 能動的アクション発生時にLive APIセッションを一時的に接続し、AIに発話させる。発話完了後にセッション切断 |
| 無料枠配慮 | 能動的アクションにもセッション開始回数がカウントされる。1日の能動的アクション上限（初期値: 10回）を`config.js`で設定可能 |
| ON/OFF | ハンバーガーメニューで能動的アクションの有効/無効を切り替え（デフォルト: ON） |

#### 2.9.2 ユーザー存在検知

アバターが独り言にならないよう、能動的アクション前にユーザーが画面前にいるかを判定する。

| 検知方式 | 説明 |
| ---------- | ------ |
| マイク振幅監視 | ウェイクワード待機中の`SpeechRecognition`と並行して、`AnalyserNode`でマイク入力の振幅を監視。一定以上の音声入力（環境音・キーボード音等）があれば「ユーザー存在」と判定 |
| Page Visibility API | `document.visibilityState`が`visible`の場合のみ能動的アクションを実行。タブがバックグラウンド時は抑制 |
| 直近の対話履歴 | 過去30分以内にウェイクワード対話があった場合、ユーザー存在とみなす優先度を上げる |

※人感センサを搭載しないWebアプリの制約上、マイク音量とタブ可視性による推定検知とする。プライバシーのため、マイク音声データは振幅値のみ使用しサーバー送信しない。

#### 2.9.3 イベントトリガー

能動的アクションを発生させる条件を「イベント」として定義する。イベントは`config.js`で設定可能。

| イベント種別 | トリガー条件 | 発話例 |
| -------------- | -------------- | -------- |
| 時間帯挨拶 | 朝（6:00〜9:00）/ 昼（11:00〜14:00）/ 夜（18:00〜22:00）の各時間帯に1回のみ | 「おはよう！今日も一日頑張ろうね」 / 「お昼だよ。ちゃんと食べた？」 |
| アイドル話しかけ | 最後の対話から30分以上経過 + ユーザー存在検知時 | 「ねえ、何か話そ？最近どう？」 |
| 天気提案 | 朝の時間帯で、かつ降水確率50%以上の場合 | 「今日は雨が降りそうだから傘忘れないでね」 |

※イベント発生時はLive APIセッションを接続し、system instructionに「以下の内容を自然な会話として発話してください」とコンテキストを付与してAIに発話させる。

#### 2.9.4 シーケンス

```
[常時] ウェイクワード待機中 + ユーザー存在検知監視
         │
         ├─ イベント条件成立（例: 朝の時間帯 + ユーザー存在）
         │     │
         │     ├─ 能動的アクション上限チェック（超過時はスキップ）
         │     ├─ Live API WebSocket接続開始
         │     ├─ イベントコンテキストを送信 → AIが能動的に発話
         │     ├─ 音声再生 + リップシンク + 字幕表示
         │     └─ 発話完了後、セッション切断 → ウェイクワード待機に戻る
         │
         └─ イベント条件未成立 → ウェイクワード待機継続
```

※能動的アクション発話中にユーザーが話しかけた場合（ウェイクワード検出）、能動的発話を中断し、通常の対話セッションに移行する。

### 3.2 system instruction

アバターに応じて名前部分を動的に切り替えて生成する。

```
あなたは「{name}」という名前の3Dアバターアシスタントです。
以下のルールを守ってください：

1. 明るく親しみやすい性格で、ユーザーをフレンドリーにサポートする。
2. 一回の発言は短く（2〜3文以内）にまとめる。
3. 必要に応じてユーザーに質問を投げ返し、対話を促す。
4. 応答テキストの先頭に必ず感情タグ [neutral], [joy], [angry], [sorrow], [fun], [surprised], [greeted] のいずれかを付ける。タグは発声せずテキスト先頭にのみ付ける。
5. ユーザーが「今何時」「天気」「タイマー」「アラーム」「ポモドーロ」「最初はグー」などのキーフレーズを言った場合、対応する関数を呼び出す。

以下は応答の例です:

ユーザー: こんにちは
{name}: [greeted]こんにちは！今日もよい日だね。何かお手伝いすることある？

ユーザー: 3分タイマーかけて
{name}: [fun]3分だね、了解！すぐセットするね。終わったら教えるよ。

ユーザー: 今日どんな気分？
{name}: [joy]絶好調だよ！あなたと話せて嬉しいな。最近どう過ごしてる？

ユーザー: あー、疲れた…
{name}: [sorrow]お疲れ様…。無理しないでね。
```

※`{name}` は選択中のアバター名（リリ / エマ / ルカ）に置換される。Few-shot例は感情タグの適切な選択、2〜3文の応答長、関数呼び出しの自然な判定をモデルに学習させる目的で挿入。例は日本語ベースだが、言語切替時はsystem instructionの言語に合わせて動的生成する。

#### 5.3.3 サーバーからのメッセージ受信

| メッセージ種別 | 処理 |
| ---------------- | ------ |
| `setupComplete` | セッション確立完了。音声入力送信開始 |
| `serverContent.audioChunk` | 音声データ受信 → AudioBuffer再生 + リップシンク |
| `serverContent.outputAudioTranscription.text` | テキスト受信 → 感情タグ抽出 + 字幕表示 |
| `toolCall` | 関数呼び出し要求 → ローカルで関数実行 → `toolResponse`を送信 |
| `interrupted` | ユーザー割り込み検知 → 音声再生停止 |

#### 5.3.4 関数応答（toolResponse）

```json
{
  "toolResponse": {

    ]
  }
}
```

※`SpeechSynthesis`は音声のカスタマイズ性が低いため、あくまでフォールバック。可能な限りLive APIセッションを維持して発話させることを優先。

## 8. エラー処理

| エラー種別 | 処理 |
| ------------ | ------ |
| APIキー未設定 | ウェイクワード検出時に「APIキーを設定してください」とステータス表示。セッション開始せず |
| WebSocket接続失敗 | ステータスに「接続エラー」と表示。5秒後にウェイクワード待機に復帰 |
| Live APIセッション切断（予期せぬ） | ステータスに「セッション切断」と表示。ウェイクワード待機に復帰 |
| マイク権限拒否 | 「マイクへのアクセスを許可してください」とステータス表示 |
| Web Speech API非対応 | フォールバック: 画面タップ/ボタン押下でセッション開始するUIを表示 |
| VRMロード失敗 | 「VRMファイルの読み込みに失敗しました」とエラー表示。デフォルトアバターにフォールバック |
| 天気API取得失敗 | **【スコープ外】** `get_weather`非実装のため発生しない |
| 無料枠超過 | **【スコープ外】** 無料枠警告機能非実装のため、ステータス表示のみ省略しAPIエラーは汎用処理 |

## 9. セキュリティ・プライバシー

| 項目 | 方針 |
| ------ | ------ |
| APIキー取り扱い | ローカルストレージに保存。ハンバーガーメニューのAPIキー設定項目の上に「APIキーはローカルでのみ使用し、公開しないでください」の警告を表示 |
| マイク音声 | Live APIセッション中のみ送信。ウェイクワード待機中はローカル認識のみでサーバー送信なし。能動的アクションのユーザー存在検知で使用するマイク振幅値もサーバー送信しない |
| 位置情報 | 天気取得時のみ `navigator.geolocation` を使用。取得後はメモリに保持せず破棄 |
| HTTPS要件 | マイク・Web Speech API使用のためHTTPS必須。ローカル開発は `localhost` |

## 10. 制約・注意事項

1. **Gemini Live API仕様変更可能性**: 本仕様は2026年7月時点のAPI仕様に基づく。プロトコル・音声名・モデル名等は変更される可能性があるため、`config.js`で一元管理し容易に修正可能にする。

2. **Web Speech APIの制限**: Chrome系ブラウザのみ確実動作。Safari/Firefoxでは挙動が異なる可能性。フォールバックUIの提供を必須とする。

3. **バックグラウンドタブの制限**: ブラウザタブがバックグラウンド時、`setTimeout`の精度が低下する。タイマー・アラームは実時刻（`Date.now()`）ベースで監視し、フォアグラウンド復帰時に補正する。

4. **VRMAアニメーションのライセンス**: VRMA_01〜07ファイルはピクシブ株式会社VRoidプロジェクトの著作物。クレジット表記「キャラクターアニメーション: ピクシブ株式会社 VRoidプロジェクト」をアプリ内のAbout/クレジット画面に含める。001〜008のモーションパックはへすい/rerofumi氏制作のCC0（パブリックドメイン）でありクレジット表記不要だが、任意の表記を推奨。

5. **音声の自然さ**: `SpeechSynthesis` APIの音声は機械的になりがち。フォールバック時はテキストを簡潔にし、Live API経由を最優先とする。

6. **無料枠の正確な把握**: Gemini Live APIの無料枠制限は時期によって変動する。`config.js`の閾値は目安であり、実際の制限はGoogleのドキュメントで最新確認が必要。
