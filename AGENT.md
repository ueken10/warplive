# 開発エージェント向け指示 (AGENT)

本プロジェクトにおいて、AI（GitHub Copilot等）がコード生成・ファイル修正・技術提案を行う際に遵守しなければならないシステムプロンプト的な絶対ルールです。

## 1. 言語の選定: Vanilla JavaScript の徹底

* **TypeScript は絶対に使用しないでください。**
* 今後作成するすべてのスクリプトファイルは必ず `.js` 拡張子とし、TypeScript固有の型構文（インターフェース、型アノテーションなど）を含めてはいけません。
* 関数やクラスに対して型情報を付与してエディタの補完を効かせたい場合は、すべて **JSDoc** 形式（`/** @type {string} */` 等）で行ってください。

## 2. ビルドツールの完全排除: Vite等の不使用

* **Vite、Webpack、ESBuild 等のモジュールバンドラーやビルドツールを一切使用・提案しないでください。**
* プロジェクトは「ブラウザにネイティブな ES Modules (`<script type="module">` と `import/export`) を直接読み込ませて動作する」完全な静的構成をとります。

## 3. 外部依存モジュールの解決方法

* 新しいライブラリパッケージを導入する際、`npm install` コマンドを使用することは可能ですが、それをJSファイル内で単に `import { X } from 'パッケージ名';` と記述することはできません（バンドラーが存在しないためエラーになります）。
* 外部ライブラリを使用する場合は以下いずれかの方法を提案してください。
  1. CDNからの直接読み込み（例: `import { X } from 'https://esm.sh/package-name';` や `jsdelivr` 等）
  2. `node_modules` 内のビルド済みブラウザ用ファイルパスへの相対パス直接指定
  3. `package.json` のスクリプトなどを用いて、ビルド済みスクリプトを所定のフォルダへコピーしてから相対パスで読み込む手法

## 4. 全体方針

* 提案するコード片はすべて「事前コンパイルやトランスパイル無しで、最新のブラウザのV8エンジン/JavaScriptエンジンでそのまま解釈・実行できるコード」であることを必ず確認してください。
